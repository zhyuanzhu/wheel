# CDN版本号的更新

- 缓存更新

  - 利用 CDN 或 nginx 后台刷新文件路径，更新文件 header 头；发生线上 bug 或者已上线，用户看到的可能还是旧信息

- 文件 name.v1 - vxxx.js

  - 大功能每次更新迭代一个大版本，比如从 v1 - v2

  - 小功能每次增加 0.0.1 或 0.1；项目如果频繁上线，会有多个版本

  - 在某一时间点，例如年末，可以利用 nginx 统一配置，302 指向到最新的版本

- 时间戳.文件name.js

  - 以每次上线的时间点做差异

- 文件 hash.文件name.js

  - 以文件内容 hash 做 key

  - 每次上线，文件的路径会不一致
